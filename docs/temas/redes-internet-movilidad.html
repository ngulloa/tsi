Redes, Internet y movilidad

Guía conceptual sobre redes, Internet, servicios digitales y tecnologías móviles en sistemas de información.

Las redes permiten que personas, dispositivos, aplicaciones y datos trabajen como un sistema distribuido. No son solo soporte técnico: condicionan la continuidad, velocidad, seguridad y alcance real de los sistemas de información.

La clave es distinguir infraestructura de red, servicios de Internet y movilidad. Una aplicación puede estar bien diseñada, pero si falla la conectividad, la seguridad o el acceso móvil, el proceso organizacional también falla.

Definiciones clave

  • Red de telecomunicaciones: conjunto de dispositivos, medios, reglas y servicios que permiten transmitir datos, voz, video o señales entre puntos.
  • Conectividad: capacidad de usuarios, dispositivos, aplicaciones y datos para comunicarse mediante redes.
  • Cliente/servidor: modelo donde un cliente solicita servicios o datos y un servidor los entrega bajo reglas de acceso.
  • Conmutación de paquetes: forma de transmitir mensajes dividiéndolos en paquetes que viajan por la red y se reensamblan al llegar.
  • Protocolo: regla común que define cómo se formatean, envían, reciben o interpretan los datos.
  • TCP/IP: familia de protocolos que permite conectar redes distintas y sostener el funcionamiento de Internet.
  • Dirección IP: identificador lógico que permite ubicar un dispositivo o servicio dentro de una red.
  • DNS: sistema que traduce nombres de dominio legibles por personas a direcciones IP.
  • Ancho de banda: capacidad de transmisión de datos de un enlace durante un periodo.
  • Latencia: demora percibida entre una solicitud y una respuesta.
  • Red de área local: red que conecta dispositivos dentro de una oficina, sala, edificio o campus cercano.
  • Red de área amplia: red que conecta ubicaciones distantes, normalmente usando enlaces de operadores o Internet.
  • Internet: red global de redes que conecta millones de redes mediante estándares comunes.
  • World Wide Web: servicio de Internet basado en páginas, enlaces, navegadores y protocolos como HTTP.
  • Intranet: red o sitio interno usado por una organización para compartir información y servicios privados.
  • Extranet: extensión controlada de una intranet para socios, proveedores, clientes u otros actores autorizados.
  • Proveedor de servicios de Internet: empresa u organización que entrega acceso a Internet.
  • Wi-Fi: tecnología de red inalámbrica local basada en estándares 802.11.
  • Bluetooth: tecnología inalámbrica de corto alcance para conectar dispositivos personales o periféricos.
  • Red celular: infraestructura móvil que conecta teléfonos y dispositivos mediante antenas y operadores.
  • VoIP: transmisión de voz como datos digitales sobre redes IP.
  • VPN: conexión privada y cifrada creada sobre una red pública como Internet.
  • RFID: identificación por radiofrecuencia para reconocer objetos, productos, personas o activos mediante etiquetas y lectores.
  • Redes de sensores inalámbricas: redes de nodos que capturan datos del entorno y los envían para análisis.
  • Internet de las cosas: conexión de objetos físicos con sensores, software y redes para monitorear o actuar sobre el mundo real.
  • Seguridad de red: controles para proteger comunicaciones, dispositivos y servicios contra accesos, fallas o usos no autorizados.

La red como infraestructura del sistema

Una red de telecomunicaciones conecta componentes técnicos, pero su impacto es organizacional. Permite coordinar procesos, compartir información, atender usuarios, operar a distancia y sostener servicios en línea.

En un sistema de información, la red influye en:

  • Disponibilidad: si la red cae, el servicio puede quedar inaccesible.
  • Velocidad: si hay poco ancho de banda o mucha latencia, la experiencia se vuelve lenta.
  • Alcance: una red móvil o de Internet permite operar fuera de una oficina.
  • Seguridad: más conexiones también significan más puntos de acceso que controlar.
  • Integración: redes comunes permiten que aplicaciones, bases de datos y usuarios trabajen sobre la misma información.

Una municipalidad que recibe solicitudes en terreno necesita celulares, red móvil, acceso seguro al sistema, sincronización de datos y soporte si la conexión falla. La red no es un accesorio: define si el proceso puede hacerse fuera de la oficina.

Protocolos, paquetes y direccionamiento

Las redes funcionan porque los equipos siguen reglas compartidas. Un protocolo define cómo se envían los datos, cómo se identifican los equipos, cómo se corrigen errores y cómo se interpreta una solicitud.

La conmutación de paquetes divide un mensaje en unidades pequeñas. Cada paquete lleva datos y señales de control para llegar a destino. Esto permite aprovechar mejor la capacidad de la red, porque diferentes comunicaciones pueden compartir caminos.

Idea simple: enviar un archivo por Internet no es como mandar una caja cerrada por un único camino. Es más parecido a dividir el contenido en sobres pequeños, enviarlos por rutas posibles y ordenarlos al llegar.

TCP/IP es central porque permite que redes distintas se comuniquen. En una explicación de prueba, basta con reconocer cuatro niveles:

  • Aplicación: servicios usados por programas, como Web o correo.
  • Transporte: entrega de datos entre origen y destino, con control de comunicación.
  • Internet: direccionamiento y enrutamiento mediante IP.
  • Interfaz de red: transmisión física o inalámbrica en el medio disponible.

La dirección IP identifica equipos o servicios en la red. El DNS evita que las personas memoricen números: permite escribir un dominio y llegar a la dirección correspondiente.

Tipos de red por alcance

Las redes se clasifican de forma útil según el área que cubren:

  • PAN: red personal de muy corto alcance, por ejemplo un teléfono conectado a audífonos por Bluetooth.
  • LAN: red de área local, como la red de una oficina, laboratorio o edificio.
  • MAN: red metropolitana, pensada para cubrir una ciudad o zona urbana.
  • WAN: red de área amplia, usada para conectar sedes distantes o redes de gran escala.

No memorices solo siglas. Pregunta siempre: ¿qué distancia cubre?, ¿quién la administra?, ¿qué dispositivos conecta? y ¿qué proceso depende de ella?

También es útil distinguir dos formas de organización:

  • Cliente/servidor: un servidor centraliza servicios, datos o reglas de acceso para clientes.
  • Igual a igual: varios equipos comparten recursos directamente sin depender de un servidor central.

Internet, Web y servicios

Internet es la red global de redes. La World Wide Web es un servicio que funciona sobre Internet usando navegadores, páginas, enlaces y protocolos como HTTP.

Internet no es lo mismo que Web. Internet también permite correo, mensajería, videollamadas, transferencia de archivos, servicios en la nube, VPN, VoIP y muchas otras comunicaciones.

Servicios importantes:

  • Correo y mensajería: comunicación asincrónica o inmediata entre usuarios.
  • Web: acceso a páginas, plataformas y aplicaciones mediante navegador.
  • VoIP: voz transmitida como paquetes de datos.
  • VPN: acceso cifrado a recursos privados usando Internet como transporte.
  • Servicios en la nube: aplicaciones, almacenamiento o infraestructura entregados a través de red.
  • Intranets y extranets: servicios privados para empleados o actores externos autorizados.

Una universidad puede usar Internet para clases virtuales, una intranet para documentos internos, una extranet para proveedores y una VPN para que funcionarios entren a sistemas administrativos desde fuera del campus.

Movilidad y redes inalámbricas

La movilidad agrega una exigencia: el usuario espera acceder a información y servicios desde cualquier lugar razonable, con dispositivos portátiles y redes variables.

Tecnologías frecuentes:

  • Wi-Fi: conecta dispositivos a una red local inalámbrica; depende de puntos de acceso y cobertura cercana.
  • Bluetooth: conecta dispositivos personales de corto alcance; sirve para periféricos, sensores o transferencia cercana.
  • Red celular: permite acceso móvil en áreas amplias usando antenas y operadores.
  • RFID: identifica objetos o activos mediante etiquetas y lectores.
  • Redes de sensores inalámbricas: capturan datos del entorno, como temperatura, ubicación, movimiento o estado de equipos.
  • Internet de las cosas: integra objetos físicos conectados a procesos digitales.

En logística, RFID puede registrar el paso de productos por bodega, sensores pueden medir temperatura y la red celular puede informar el estado de un despacho desde un camión.

La movilidad mejora cobertura operativa, pero también aumenta riesgos: pérdida de dispositivos, conexiones inseguras, baja señal, datos sensibles fuera de la oficina y necesidad de sincronizar información.

Seguridad, continuidad y gestión

Mientras más conectado está un sistema, más depende de controles técnicos y reglas organizacionales. La seguridad de red busca proteger comunicaciones, usuarios y servicios.

Medidas típicas:

  • Autenticación de usuarios y dispositivos.
  • Cifrado de comunicaciones, especialmente fuera de la red interna.
  • Segmentación de redes para separar invitados, empleados, servidores y sistemas críticos.
  • Uso de VPN cuando se accede a recursos privados desde Internet.
  • Monitoreo de tráfico, disponibilidad y eventos sospechosos.
  • Planes de respaldo o continuidad si falla el proveedor, la energía o la conexión.

Una red rápida pero sin controles puede ser un riesgo. Una red segura pero sin continuidad puede detener operaciones. En casos, evalúa velocidad, acceso, protección y dependencia del proceso.

Comparaciones importantes

Internet vs. Web: Internet es la infraestructura global; la Web es uno de sus servicios.

LAN vs. WAN: LAN conecta un espacio cercano; WAN conecta ubicaciones distantes.

Wi-Fi vs. red celular: Wi-Fi suele depender de un punto de acceso local; la red celular depende de antenas y operador móvil.

Bluetooth vs. Wi-Fi: Bluetooth sirve para corto alcance y bajo consumo; Wi-Fi sirve para acceso local a red o Internet con más capacidad.

Intranet vs. extranet: la intranet es interna; la extranet abre acceso controlado a externos autorizados.

VPN vs. Internet público: la VPN usa Internet como transporte, pero crea un canal cifrado hacia recursos privados.

Ancho de banda vs. latencia: ancho de banda es capacidad; latencia es demora. Una videollamada puede fallar por cualquiera de los dos.

RFID vs. código de barras: RFID puede leerse por radio y no requiere la misma línea visual directa; el código de barras exige escaneo visible.

Observaciones de prueba

  • Muy probable: explicar por qué una red es parte del sistema de información.
  • Muy probable: diferenciar Internet y Web.
  • Diferencia infraestructura, protocolos y servicios.
  • TCP/IP se entiende como estándar que permite interoperabilidad.
  • LAN, WAN, Wi-Fi, Bluetooth y red celular no son sinónimos.
  • Una VPN no reemplaza Internet: crea un canal seguro sobre una red pública.
  • La movilidad aporta acceso ubicuo, pero aumenta exigencias de seguridad y continuidad.
  • RFID y sensores conectan objetos físicos con sistemas de información.
  • En un caso, no basta con decir “falta Internet”; identifica proceso afectado, impacto y medida concreta.

Casos breves

Atención en terreno: inspectores registran solicitudes desde teléfonos. Si la señal móvil falla, deben guardar datos temporalmente y sincronizar después. El caso mezcla movilidad, continuidad y seguridad.

Wi-Fi institucional: una sede ofrece red para estudiantes, funcionarios e invitados. Conviene separar accesos para proteger sistemas internos y evitar que una red pública afecte servicios críticos.

Teletrabajo administrativo: funcionarios acceden desde casa a sistemas internos. La VPN, autenticación y permisos reducen riesgos al usar Internet como medio de conexión.

Inventario con RFID: una bodega registra entradas y salidas de productos con etiquetas. El beneficio es visibilidad rápida; el desafío es integrar lectores, red, datos y procesos de reposición.

Videollamada de soporte: una reunión puede fallar aunque “haya Internet” si existe poca capacidad, alta latencia, interferencia Wi-Fi o saturación de la red.

Preguntas conceptuales

Pregunta 1: ¿Por qué una red es parte del sistema de información y no solo soporte técnico?

Porque permite que personas, procesos, aplicaciones y datos se conecten. Si la red falla, el sistema puede dejar de apoyar decisiones, atención o coordinación.

Pregunta 2: ¿Qué componentes básicos necesita una red sencilla?

Necesita dispositivos, interfaces de red, un medio de conexión, reglas o software de comunicación y equipos que dirijan el tráfico, como switches, puntos de acceso o routers.

Pregunta 3: ¿Qué función cumple un protocolo?

Define reglas comunes para que los equipos sepan cómo enviar, recibir, ordenar e interpretar datos.

Pregunta 4: ¿Qué es la conmutación de paquetes?

Es dividir un mensaje en paquetes pequeños que viajan por la red y se vuelven a ordenar al llegar a destino.

Pregunta 5: ¿Por qué TCP/IP es importante?

Porque entrega un conjunto común de protocolos que permite conectar redes y equipos distintos, haciendo posible Internet y muchos servicios asociados.

Pregunta 6: ¿Qué diferencia hay entre Internet y la Web?

Internet es la red global de redes. La Web es un servicio que usa Internet para acceder a páginas, enlaces y aplicaciones mediante navegadores.

Pregunta 7: ¿Qué problema resuelve DNS?

Permite usar nombres de dominio fáciles de recordar y traducirlos a direcciones IP que los equipos pueden localizar.

Pregunta 8: ¿Qué diferencia hay entre ancho de banda y latencia?

El ancho de banda indica capacidad de transmisión. La latencia indica demora entre solicitud y respuesta.

Pregunta 9: ¿Qué diferencia hay entre LAN y WAN?

Una LAN conecta dispositivos en un espacio cercano, como una oficina. Una WAN conecta ubicaciones distantes, como sedes de una organización.

Pregunta 10: ¿Qué diferencia hay entre cliente/servidor e igual a igual?

En cliente/servidor, un servidor entrega servicios o datos a clientes. En igual a igual, los equipos comparten recursos directamente entre sí.

Pregunta 11: ¿Qué diferencia hay entre intranet y extranet?

La intranet es privada para uso interno. La extranet abre acceso controlado a externos autorizados, como proveedores o clientes.

Pregunta 12: ¿Para qué sirve una VPN?

Sirve para crear una conexión cifrada y privada sobre una red pública, normalmente para acceder de forma segura a recursos internos.

Pregunta 13: ¿Qué es VoIP?

Es la transmisión de voz como datos digitales sobre redes IP, por ejemplo en telefonía por Internet o llamadas de aplicaciones.

Pregunta 14: ¿Qué diferencia hay entre Wi-Fi y Bluetooth?

Wi-Fi se usa para conectar dispositivos a una red local o a Internet. Bluetooth se usa para conexiones personales de corto alcance y bajo consumo.

Pregunta 15: ¿Qué aporta una red celular frente a una red Wi-Fi?

Aporta movilidad en áreas amplias mediante antenas y operadores, mientras que Wi-Fi depende de cobertura local cercana.

Pregunta 16: ¿Qué permite RFID en una organización?

Permite identificar y rastrear objetos, productos, activos o personas mediante etiquetas y lectores de radiofrecuencia.

Pregunta 17: ¿Qué relación hay entre sensores inalámbricos e Internet de las cosas?

Los sensores capturan datos del mundo físico y, conectados por redes, permiten que objetos y procesos formen parte de sistemas digitales.

Pregunta 18: ¿Por qué la conectividad puede convertirse en un riesgo operativo?

Porque si procesos críticos dependen de la red, una falla de conexión, seguridad o proveedor puede detener atención, ventas o coordinación interna.

Pregunta 19: ¿Qué cambia cuando los usuarios acceden desde dispositivos móviles?

Cambia el contexto de uso: hay redes variables, pantallas pequeñas, riesgo de pérdida del equipo, necesidad de sincronización y mayores exigencias de seguridad.

Pregunta 20: ¿Cómo conviene analizar un caso de redes en prueba?

Identificando el proceso afectado, tipo de red, usuarios, servicios dependientes, riesgos de seguridad, impacto de una falla y medidas concretas de continuidad.
